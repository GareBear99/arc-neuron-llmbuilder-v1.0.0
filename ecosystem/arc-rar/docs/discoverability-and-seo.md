# Discoverability and Findability

This is the honest version: you cannot guarantee discovery, but you can maximize it.

## Highest-leverage findability actions

1. Use the exact searchable terms in README and site copy:
   - WinRAR alternative
   - archive manager
   - RAR extractor
   - ZIP creator
   - 7z support
   - native macOS archive app
   - native Windows archive app
   - native Linux archive app
   - CLI archive tool
   - terminal-driven GUI archive manager

2. Publish examples that match search intent:
   - “extract RAR from terminal”
   - “open archive from command line”
   - “control archive GUI from script”
   - “native alternative to WinRAR for macOS/Linux”

3. Create rich docs and screenshots:
   - README hero image
   - OS-specific pages
   - command examples
   - shell integration screenshots
   - custom OS porting page

4. Package broadly:
   - GitHub releases
   - Homebrew tap
   - winget manifest
   - scoop/choco package
   - AUR / distro package references

5. Use stable naming everywhere:
   - product name Arc-RAR
   - binary name `arc-rar`
   - docs titles with exact keywords

## Things not to claim
- “guaranteed SEO”
- “best app” without evidence
- “RAR creator” unless you have licensed that ability
