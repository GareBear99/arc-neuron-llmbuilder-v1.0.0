# Implementation Matrix

This file is the plain-language truth surface for the current repository state.
It is meant to separate what is already live from what is bounded, optional, or still doctrine-driven.

## Status legend

- **Live**: implemented in repo and covered by direct tests or CLI flows
- **Bounded**: implemented as a constrained adapter or scaffold, but not equivalent to field-hardened production capability
- **Optional**: install/use is not required for core runtime value
- **Experimental**: directionally useful, but should not be presented as finished autonomy
- **Planned**: doctrine target, not yet claimed as implemented
