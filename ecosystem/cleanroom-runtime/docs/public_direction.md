# Public Direction Goals

This repository is being positioned as a real autonomy foundation with an honest growth path.

## Direction

1. Keep the base runtime installable and useful without heavyweight multimodal or robotics dependencies.
2. Support future vision, audio, simulation, desktop-control, and robotics through optional adapters.
3. Preserve the repo's strongest differentiators: receipts, replay, rollback, memory lineage, repair lineage, and bounded self-improvement.
4. Use local GGUF or replaceable model cores as planners over structured state, not as uncontrolled raw-world drivers.
5. Publish the repo as a serious engineering foundation instead of overselling it as already-solved AGI.

## What success looks like

- A first-time GitHub visitor understands what the system is in under two minutes.
- A Python developer can install and run the base runtime without camera or robot stacks.
- An advanced builder can later attach perception or embodiment layers without rewriting the core.
- The repo keeps compounding trust through tests, docs, traces, demos, and measured evolution.
