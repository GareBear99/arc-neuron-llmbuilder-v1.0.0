# Ecosystem Package Audit — Integration Use Map

## Cleanroom Runtime
Use for training signal and benchmark realism.
- Train: planning posture, critique posture, bounded repair, rollback language, evidence-first behavior
- Benchmark: repair realism, constraint preservation, cautious patching, calibration
- Keep external: runtime implementation

## ARC-Core
Use for state/evidence/authority structure.
- Train: fact vs inference, proposal before action, evidence-linked reasoning, authority gating
- Benchmark: state transitions, proposal correctness, refusal without authority
- Keep external: system of record runtime and state engine

## OmniBinary Runtime
Use for execution-lane reasoning.
- Train: inspect before act, lane selection, blocker honesty, environment awareness
- Benchmark: execution lane selection, preview vs execute honesty, anti-overclaim tasks
- Keep external: actual runtime/translation engine

## Arc-RAR
Use for receipt/archive discipline.
- Train: archive before mutate, receipt completeness, replay lineage, restore logic
- Benchmark: archive/restore reasoning, receipt completeness, artifact lineage
- Keep external: app/runtime implementation

## ARC Language Module
Use selectively.
- Train: language-boundary honesty, transliteration caution, low-resource caution
- Retrieval: large language/domain packs, provider inventories, evolving knowledge
- Keep external: full language subsystem and datasets

## AI Screenshot Attachment
Use as a tool integration.
- Tool: screenshot capture adapter
- Train minimally: evidence attachment awareness
- Keep external: shell capture implementation

## AI Command Keyboard
Use lightly.
- Use mainly as UI/operator inspiration and intent mapping reference
- Not a central cognition-core training source

## Claude Code source archive
Use as benchmark pressure, not raw weight stuffing.
- Benchmark: longer workflow tasks, codebase surgery realism, tool-chain realism
- Retrieval: selected workflow patterns only
- Keep external: full codebase

## Final integration rule
Cognition Core should absorb:
- curated cognitive habits
- task/eval pressure
- tool contracts
- receipt/evidence structure

It should not absorb:
- raw monolithic code from the other repos
- entire external runtimes as duplicated internal subsystems
