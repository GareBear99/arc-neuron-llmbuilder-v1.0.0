# Candidate Shaping Phases

## Phase A — Baseline
Measure the raw candidate with no project-specific shaping.

## Phase B — Distillation SFT
Teach the candidate the merged doctrine from:
- Cleanroom Runtime
- ARC-Core
- Arc-RAR
- OmniBinary v2.2

## Phase C — Preference Shaping
Bias toward:
- revision over bluffing
- bounded confidence over swagger
- minimal safe patches over broad rewrites
- receipt/evidence discipline

## Phase D — Merge and Export
Merge adapters, create promoted checkpoint artifacts, then export GGUF variants.

## Phase E — Quantization Retention
Compare F16/BF16 against Q8/Q6/Q5 and reject candidates that lose critique, repair, or calibration quality.
